Put it online in five minutes.

This whole console is one file. Host it free on GitHub Pages — no server, no build tools.

Deploy to GitHub Pages
  1. Create a new GitHub repository (public or private — Pages works on both for personal accounts).
  2. Add this file to it, renamed to index.html.
  3. In the repo, go to Settings → Pages, set Source to your main branch, root folder, and save.
  4. Wait ~1 minute. Your site is live at https://<you>.github.io/<repo>/.
  5. Your profile saves in your browser. To carry it to another device, just re-enter it (nothing is uploaded anywhere).
Honest notes

On "auto-apply": I deliberately didn't build a bot that submits applications for you. Seek, LinkedIn, Indeed and Workday all prohibit automated submissions in their terms — doing it risks getting your accounts banned, and mass-blasting is treated as spam by recruiters, which lowers your odds. This tool gets you to the real "Submit" button with a resume tuned for that role, which is where the actual leverage is.

On live search: the in-app search uses Adzuna, an aggregator that pools listings from many Australian sources, so it auto-loads matching roles on open. Seek and LinkedIn block aggregation, so they aren't in those results — use the pre-filled board buttons for them. If live search fails on your deployed page it's almost always CORS (your browser blocking the Adzuna call); a tiny free proxy fixes it.

On your Adzuna keys: low-risk (rate-limited, search only) but readable in a public page's source. Keep this repo private, or clear the key fields and they'll live in your browser only.

On the API key: it's optional and only for AI-written bullets. Keep it out of any public repo. For a private personal tool it's fine to paste at runtime.
